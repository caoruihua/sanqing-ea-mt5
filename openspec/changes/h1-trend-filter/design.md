# H1 趋势过滤器 - 技术设计

## Context

### 当前状态

当前 EA 使用 M5 周期的 EMA 9/21 双均线系统判断交易方向：
- EMA 9 > EMA 21 → 看多
- EMA 9 < EMA 21 → 看空

**问题**：
1. EMA 9/21 都是短期均线，无法代表中长期趋势
2. 在震荡行情中频繁交叉，产生大量假信号
3. 止损距离（1.2 × ATR）过紧，容易被正常波动触及
4. ADX 过滤只在部分策略中应用，不一致

### 约束条件

- 保持现有策略逻辑不变，只增加过滤层
- 新增参数需要有合理默认值
- 不能显著影响 EA 执行性能
- 保持代码可维护性

## Goals / Non-Goals

**Goals:**
- 增加 H1 周期 EMA50 作为中期趋势方向过滤器
- 统一所有策略的 ADX 趋势强度过滤
- 放宽止损距离至 1.8 × ATR
- 减少震荡行情中的假信号和无效交易

**Non-Goals:**
- 不修改策略的核心入场逻辑
- 不增加 H4 或更大周期的过滤（对短线交易太慢）
- 不修改保护引擎的止损移动逻辑
- 不改变现有的盈亏比设置

## Decisions

### 决策 1：使用 H1 EMA50 作为趋势过滤器

**选择**: H1 EMA50

**备选方案**:
| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| H1 EMA50 | 2天趋势加权平均，反应适中 | - | ✅ 选择 |
| H1 EMA200 | 更稳定 | 滞后太久，错过入场时机 | ❌ 放弃 |
| H4 EMA50 | 更稳定 | 对短线交易太慢 | ❌ 放弃 |
| H1 EMA20+EMA50 双均线 | 更精确 | 复杂度高，增加假信号 | ❌ 放弃 |

**理由**: H1 EMA50 代表过去 50 小时（约 2 天）的价格加权平均，对 M5 短线交易来说是合适的中期趋势参考。

### 决策 2：趋势过滤规则

**规则**:
```
H1 Close > EMA50 → 只允许做多（禁止做空）
H1 Close < EMA50 → 只允许做空（禁止做多）
```

**备选方案**:
| 方案 | 说明 | 结论 |
|------|------|------|
| 严格方向过滤 | H1 方向决定可交易方向 | ✅ 选择 |
| 软性方向过滤 | H1 方向作为评分因子 | ❌ 复杂度高 |
| 双向都可交易 | 无方向过滤 | ❌ 不解决问题 |

**理由**: 简单明确的规则，易于理解和调试，有效过滤逆势交易。

### 决策 3：ADX 过滤统一

**选择**: 所有趋势跟随策略统一使用 ADX > 25 过滤

**当前状态**:
| 策略 | ADX 过滤 |
|------|---------|
| TrendContinuation | ✅ 有 |
| ExpansionFollow | ✅ 有 |
| Pullback | ❌ 无 |
| Reversal | ❌ 无 |

**变更后**:
| 策略 | ADX 过滤 | 说明 |
|------|---------|------|
| TrendContinuation | ✅ 有 | 保持 |
| ExpansionFollow | ✅ 有 | 保持 |
| Pullback | ✅ 新增 | 避免震荡中回调失败 |
| Reversal | ✅ 新增 | 避免震荡中假反转 |

### 决策 4：止损距离调整

**选择**: 初始止损从 1.2 × ATR 调整为 1.8 × ATR

**计算依据**:
```
黄金 ATR ≈ 15-20 美元
当前止损: 1.2 × 17 = 20.4 美元
新止损:   1.8 × 17 = 30.6 美元

增加约 50% 的止损空间，容纳正常波动
```

**盈亏比保持**:
```
止损: 1.8 × ATR
止盈: 2.5 × ATR (同步调整)
盈亏比: 1.4 : 1
```

### 决策 5：Reversal 策略特殊处理

**选择**: Reversal 策略应用 ADX 过滤，但不强制应用 H1 方向过滤

**理由**:
- Reversal 策略本身就是抓反转点，需要灵活性
- ADX 过滤可以避免在无趋势震荡中假反转
- H1 方向过滤可选（通过参数控制）

## Risks / Trade-offs

### 风险 1：错过部分有效信号

**风险**: H1 过滤可能过滤掉一些短期逆势但有效的交易机会

**缓解**: 
- 可通过参数关闭 H1 过滤
- Reversal 策略不强制应用 H1 过滤

### 风险 2：H1 EMA50 在震荡中也会反复穿越

**风险**: 长时间震荡时，价格可能在 EMA50 上下反复，仍会产生假信号

**缓解**: 
- ADX > 25 过滤确保只在有趋势时交易
- 震荡时 ADX < 25，自动禁止交易

### 风险 3：止损放宽导致单笔亏损增加

**风险**: 止损从 1.2 ATR 放宽到 1.8 ATR，单笔亏损增加 50%

**缓解**: 
- 减少假信号被止损的次数
- 整体盈亏比保持或改善
- 可通过参数调整止损倍数

### 风险 4：多周期数据获取延迟

**风险**: 获取 H1 数据可能增加执行延迟

**缓解**: 
- 使用 MQL5 内置的 iMA 函数，性能开销极小
- 只在需要时计算，不缓存大量历史数据

## Migration Plan

### 实施步骤

1. **阶段 1：参数和数据结构**
   - 在 Common.mqh 中添加新参数和结构字段
   - 在 ContextBuilder.mqh 中添加 H1 数据获取

2. **阶段 2：策略修改**
   - 修改 TrendContinuationStrategy
   - 修改 PullbackStrategy
   - 修改 ExpansionFollowStrategy
   - 修改 ReversalStrategy

3. **阶段 3：测试验证**
   - 回测验证
   - 模拟盘测试
   - 实盘小仓位测试

### 回滚策略

- 设置 `InpTrendTimeframe = 0` 可禁用 H1 过滤
- 设置 `InpEmaTrendPeriod = 21` 可回退到类似 M5 EMA21 的行为
- 止损参数可通过输入参数调整

## Open Questions

1. **Reversal 策略是否需要 H1 过滤？**
   - 当前设计：可选（通过参数控制）
   - 待确认：用户反馈后决定

2. **是否需要增加 EMA 斜率过滤？**
   - 当前设计：不包含
   - 后续可考虑：如果震荡问题仍存在，可增加 EMA 斜率判断

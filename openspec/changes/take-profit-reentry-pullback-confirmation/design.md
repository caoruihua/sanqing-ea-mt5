## Context

`mt5/剥头皮脚本.mq5` 当前已有止损后冷却，但没有止盈后的再入场节奏控制。止盈平仓后，只要空仓且趋势条件仍满足，EA 可能很快再次开同方向单，容易追在短线波段末端。

目标是在现有信号链路后增加一个“止盈后回调再冲确认”过滤器：

```text
原策略信号成立
  -> 结构反转过滤
  -> 止盈后回调再冲确认
  -> 下单
```

## Goals / Non-Goals

**Goals:**

- 上一单止盈后，禁止立刻开同方向下一单。
- 多单止盈后，先等价格回调 2 美元，再从回调低点上涨 1 美元，才允许继续做多。
- 空单止盈后，先等价格反弹 2 美元，再从反弹高点下跌 1 美元，才允许继续做空。
- 趋势方向变化或等待超时时自动重置状态。
- 保留现有止损冷却、日内风控、时间控制、结构反转过滤和订单发送逻辑。

**Non-Goals:**

- 不新增反手交易。
- 不改变止盈/止损价格计算。
- 不改变已有趋势、回调、强趋势、持续趋势判断。
- 不替代结构性反转危险区过滤。

## Decisions

### 1. 使用止盈后状态机

状态机：

```text
NONE
  -> 检测到上一单止盈
WAIT_PULLBACK
  -> 回调/反弹达到 2 美元
WAIT_RESUME
  -> 顺势重新启动达到 1 美元
READY
  -> 允许下一单并重置状态
```

原因：只用固定时间冷却不够精确，价格可能还在高位/低位；状态机能要求价格先调整，再确认顺势恢复。

### 2. 用固定美元距离作为默认规则

默认参数：

```text
TakeProfitReentryPullbackUsd = 2.0
TakeProfitReentryResumeUsd   = 1.0
TakeProfitReentryMaxWaitSec  = 600
```

原因：黄金用美元距离直观，便于调参和看日志。

备选：使用 ATR 倍数。暂不采用，避免新增过多参数和理解成本。

### 3. 只限制同方向再入场

多单止盈后只限制下一次做多；空单止盈后只限制下一次做空。

如果趋势方向已经反向，状态应重置，让原有趋势逻辑自行决定。

### 4. 止损冷却优先

已有止损冷却继续保持更高优先级：

```text
止损冷却中 -> 不交易
止损冷却结束 -> 再看止盈后再入场状态
```

### 5. 检测止盈应优先使用 DEAL_REASON_TP

优先判断历史成交的 `DEAL_REASON_TP`。如果环境中原因字段不可用，再用 `profit > 0` 作为保守兜底，但日志需要说明。

## Risks / Trade-offs

- 交易频率下降 -> 这是预期效果，用于减少止盈后连追。
- 趋势很强时可能错过连续行情 -> 用 10 分钟超时重置降低长期卡住风险。
- 固定 2 美元/1 美元不适合所有波动环境 -> 先做成输入参数，后续可按回测调整。
- 历史成交原因不稳定 -> 优先 `DEAL_REASON_TP`，兜底用盈利平仓判断。
- 状态卡住 -> 趋势反向、超时、完成再入场确认时都重置。
